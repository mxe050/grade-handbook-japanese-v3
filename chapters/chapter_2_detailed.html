Chapter 2：GRADEの開発方法論 ― Core GRADEの誕生

💡 Core Concept

GRADEは2004年の導入以来、世界標準として進化を続けてきましたが、20年以上にわたる改訂により40以上のガイダンス論文が蓄積し、フレームワークが過度に複雑化しました。Core GRADEは、この複雑化の問題を解決するために、GRADEの核心的な専門家たちが「できるだけシンプルに、しかしシンプルすぎない」という原則のもとに開発した、実践的な再構築版です。

GRADEワーキンググループと Core GRADE の関係

GRADE（Grading of Recommendations Assessment, Development and Evaluation）は、エビデンスの確実性を評価し、推奨の強さを決定するための世界的な標準システムです。Cochrane CollaborationによるすべてのシステマティックレビューはGRADEに従うことが義務付けられており、WHO、American College of Physicians、UpToDate、The BMJを含む世界中の120以上の組織によって採用されています。

Core GRADEは、GRADEワーキンググループの「公式」な成果物ではありません。しかし、GRADEを開発した第一人者たちによる共同作業として誕生しました。[1]

Core GRADEの著者たちは、数百のシステマティックレビューや臨床ガイドラインの監督支援を提供し、BMJ Rapid Recommendationsの開発やWHO COVIDガイドラインの作成に貢献してきました。20年以上にわたる学生やガイドラインパネルメンバーへのGRADE教育経験が、Core GRADEの設計に反映されています。

なぜ Core GRADE が必要になったのか？

GRADEの成功と発展に伴い、以下の3つの深刻な問題が浮上しました。

1. 複雑さの増大

GRADEはますます複雑になり、新規ユーザーだけでなく既存ユーザーにとっても扱いにくいものとなっていました。特にCochrane RoB2やROBINS-Iなどのツールは、方法論的に優れているものの実用性に課題がありました。

2. 古い情報と組織化の欠如

20年以上にわたる論文の逐次的な出版により、ガイダンスが時代遅れになり、組織化されていないという問題が生じていました。ユーザーは新しいガイダンスを見つけ処理することが困難でした。

3. 重複と不一致

複数のジャーナルやプラットフォームにわたって公開された結果、かなりの重複、修正、および時代遅れの資料が混在していました。現在のガイダンスを特定するには細心の注意が必要でした。

GRADEの本質は、Journal of Clinical Epidemiology (JCE) の最初の15本のGRADEシリーズ論文に示されました。その後、「リビング（常に更新される）」なガイダンスシステムとして、以下のような重要な修正が行われてきました。

⚠️ 重要な認識

これらの更新されたガイダンスのかなりの部分は、GRADEユーザーにとって滅多に適用されないほど十分に複雑であることが判明しています。これが Core GRADE 開発の直接的な動機となりました。[2]

Core GRADE の開発原則

Core GRADEの開発にあたり、以下の基本原則が設定されました。

1
柔軟性（Flexibility）

エビデンスの確実性やエビデンスから推奨への移行については、複数の最良の判断がしばしば可能です。したがって、Core GRADEは「ルール」ではなく「ガイダンス」を提供します。際どい判断（close-call decisions）があるという認識が、この柔軟性をさらに強化します。[3]

2
実用性（Practicality）

理論的な完全性よりも、実際の臨床現場やガイドライン作成で使いやすいことを優先します。

3
概念の単純化（Simplification of Concepts）

複雑な概念をより直感的に理解できる形に再構成します。

4
厳格な取捨選択

エビデンスの確実性および推奨の方向性と強さに関する適切な判断を下すために必要不可欠でないガイダンスは、Core GRADEの一部としない。これは、ガイダンスが優雅さ、理解の深さ、透明性の向上に貢献する場合でも同様です。[4]

BMJ 7論文シリーズの構成

Core GRADEは、The BMJに掲載された7本の論文からなるシリーズとして構成されています。

論文番号タイトル主な内容
Core 1Overview全体像、PICO設定、確実性の連続体の概念
Core 2Imprecision閾値の選択、OISの簡略化、不精確さ評価のアルゴリズム
Core 3Inconsistency研究間のばらつき評価、フローチャート、サブグループ分析
Core 4Risk of BiasROBUST-RCTツール、大きな効果、用量反応、出版バイアス
Core 5Indirectness適用可能性、代理アウトカム、PICOの不一致
Core 6Summary of Findings結果の要約テーブル、標準化された表現
Core 7Evidence-to-Decisionエビデンスから推奨へ、MID、益と害のバランス

Core GRADE の主な簡素化ポイント

Core GRADEは、以下の点で従来のGRADEガイダンスを簡素化しています。

非ランダム化研究の評価

Core GRADE：当初から「確実性が低い（low）」エビデンスとして評価

従来：ROBINS-Iでは高い確実性からグレードダウンすることを許容

RCTのバイアスリスク評価

Core GRADE：シンプルなROBUST-RCTツールを使用

従来：複雑なCochrane RoB2ツールが存在

エビデンスのグレードアップ基準

Core GRADE：「大きな効果」と「用量反応勾配」の2基準のみ

従来：「もっともらしい残余交絡」の3つ目の基準も存在

EtDフレームワーク

Core GRADE：完全なチェックリスト完成は必須でなく、主要ドメインに限定

従来：より包括的なチェックリストの完成が求められる場合があった

Core GRADE の革新的要素

🗺️ フローチャートとアルゴリズム

Core GRADEにおける最も重要な革新は、詳細なフローチャートとアルゴリズムの導入です。これらは、ユーザーをGRADEのプロセスに沿ってステップバイステップで導くための視覚的なロードマップを提供します。手順を明確にし、曖昧さを低減させ、GRADEの使用をより体系的かつ利用しやすいものにします。[5]

↔️ 連続体としての確実性

Core GRADEは、エビデンスの確実性が「高・中・低・非常に低」というカテゴリーとして提示されているものの、実際には基礎となる連続体（continua）を表していることを強調しています。これは個々のドメイン評価にも全体的な確実性評価にも当てはまり、際どい判断（close calls）がいかに頻繁であるかを浮き彫りにしています。[6]
