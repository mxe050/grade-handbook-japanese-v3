Chapter 15：閾値とユーティリティ（Thresholds and Utility）

📖 Core Concept

閾値（Threshold）は、介入の効果が「臨床的に重要」と判断される境界値です。Core GRADEは、不精確さの評価とEtDフレームワークの両方で閾値を中心的な概念として位置づけています。閾値の計算にはユーティリティ（効用値）が用いられますが、ユーティリティの概念には重要な注意点があります。

閾値の種類

利益の閾値
（Benefit Threshold）

介入の効果が臨床的に重要な利益と判断される最小の効果量。MID（最小重要差）に基づいて設定されます。

害の閾値
（Harm Threshold）

介入の副作用が臨床的に重要な害と判断される最小の効果量。害のMIDに基づいて設定されます。

正味の閾値
（Net Threshold）

利益と害をユーティリティで重み付けして統合した閾値。EtDフレームワークで使用されます。

ユーティリティ（効用値）の基本概念

💡 ユーティリティとは

ユーティリティは、健康状態に対する個人の選好を0から1のスケールで表現した値です。

値意味例
1.0完全な健康健康な成人の通常の状態
0.5中程度の健康障害中等度の慢性疼痛、軽度の脳卒中後遺症
0.0死亡―

ユーティリティと重要度の違い ― 最も重要な区別

❌ 混同してはいけない2つの概念

ユーティリティ（0-1スケール）とアウトカムの重要度は、しばしば混同されますが、本質的に異なる概念です。

ユーティリティ（0-1スケール）

健康状態の「良さ」を表す。
1 = 完全な健康（最も良い状態）
0 = 死亡（最も悪い状態）

問題点：ユーティリティが高い（1に近い）ほど「良い状態」だが、それは「重要でない」ことを意味しない。

⇔
アウトカムの重要度

アウトカムが患者にとってどれだけ重要かを表す。
死亡（ユーティリティ = 0）→ 極めて重要
完全な健康（ユーティリティ = 1）→ 極めて重要

核心：0でも1でも、どちらも患者にとって極めて重要。

死亡（ユーティリティ = 0）も完全な健康（ユーティリティ = 1）も、どちらも患者にとって極めて重要です。ユーティリティのスケールは「重要度」を表しているのではなく、「健康状態の良さ」を表しています。したがって、ユーティリティのスケールと重要度のスケールは本質的に異なるものです。

ユーティリティ = 1 でそのまま計算すると閾値がおかしくなる問題

🚨 重大な計算上の問題

閾値の計算において、完全な健康状態（ユーティリティ = 1）をそのまま使用すると、閾値の位置がおかしくなります。これは、ユーティリティベースの閾値計算の根本的な問題です。[1]

なぜ問題が生じるのか

閾値計算では、アウトカムの不効用（disutility）を使用します。不効用は「完全な健康からの距離」、すなわち1 - ユーティリティで計算されます。

▼
健康状態ユーティリティ不効用（1 - U）問題
死亡0.01.0不効用が最大 → 閾値計算は正常に機能
重度の障害0.30.7不効用が大きい → 閾値計算は正常に機能
軽度の不快感0.90.1不効用が小さい → 閾値が大きくなる
完全な健康 1.0 0.0 不効用がゼロ → 閾値が無限大に！
▼

完全な健康状態のユーティリティは1.0ですが、不効用は0.0です。閾値計算式の分母に不効用が入る場合、分母がゼロになり、閾値が無限大（計算不能）になってしまいます。

これは、「完全な健康を維持すること」の重要性は非常に高いにもかかわらず、ユーティリティベースの計算ではその重要性を適切に反映できないことを意味します。

閾値の計算方法

1
不効用ベースのアプローチ（Schünemann先生）
🧮 計算式

Guidance 42では、以下の計算式を使用します：

閾値 = ベースラインリスク × (不効用アウトカム / 不効用基準)

不効用（Disutility） = 1 - ユーティリティ

例：死亡のMIDを計算する場合

  • ➡️死亡の不効用 = 1 - 0 = 1.0
  • ➡️基準アウトカム（例：入院）の不効用 = 1 - 0.6 = 0.4
  • ➡️比率 = 1.0 / 0.4 = 2.5

つまり、死亡は入院の2.5倍の重みを持つことになります。

2
パネル調査アプローチ（Guyatt先生）
👥 プロセス

Core GRADEでは、以下のプロセスでMIDを設定します：

  1. 1️⃣パネルメンバーに対して、各アウトカムのMIDの範囲を提示
  2. 2️⃣パネルメンバーが、臨床経験と患者の視点を考慮してMIDを直接設定
  3. 3️⃣パネル全体で討議し、コンセンサスを形成
  4. 4️⃣MIDの不確実性を「もっともらしい範囲」として表現
3
Generic Coefficients（汎用係数）
📊 汎用係数の例

Guidance 42では、一般的なアウトカムに対する汎用係数（generic coefficients）を提供しています。これにより、個別のユーティリティ測定が利用できない場合でも閾値を推定できます。

アウトカムユーティリティ不効用汎用係数
死亡0.001.001.00
重度の脳卒中0.200.800.80
心筋梗塞0.400.600.60
入院0.600.400.40
軽度の副作用0.900.100.10

実践上の推奨

⚠️ どちらのアプローチを使うべきか

Guyatt先生とSchünemann先生のアプローチはいずれも有効ですが、以下の点を考慮して選択してください：

  • ✅ユーティリティ値が利用可能で信頼できる場合：Schünemann先生のアプローチが再現性が高い
  • 🤔ユーティリティ値が不確実な場合：Guyatt先生のパネル調査アプローチが柔軟性が高い
  • 📈完全な健康に近いアウトカムの場合：ユーティリティ = 1の問題があるため、パネル調査アプローチが推奨
  • 🎯いずれの場合も：閾値の不確実性を認識し、「もっともらしい範囲」として幅を持たせる
✅ Chapter 15のまとめ
  • 1️⃣閾値は利益の閾値、害の閾値、正味の閾値の3種類。
  • 2️⃣ユーティリティは0（死亡）から1（完全な健康）のスケールで健康状態の「良さ」を表す。
  • 3️⃣ユーティリティと重要度は異なる概念：0（死亡）も1（完全な健康）もどちらも極めて重要。
  • 4️⃣ユーティリティ = 1でそのまま計算すると閾値がおかしくなる：不効用がゼロになり、閾値が無限大に。
  • 5️⃣Guyatt先生はパネル調査ベース、Schünemann先生はユーティリティベースのアプローチ。
  • 6️⃣閾値の不確実性を認識し、「もっともらしい範囲」として幅を持たせることが重要。
